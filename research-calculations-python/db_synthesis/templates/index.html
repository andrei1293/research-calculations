Business rules pattern for attributes Business rules pattern for entities

(A | Each) {Entity} [is described by] {attr1}, {attr2}, ..., (and) {attrN} .

Example (one attribute): Each Student is described by Full Name.

Example (two attributes): A Student is described by Last Name, and First Name.

Example (multiple attribute): Each Student is described by Full Name, Year, and Birthdate.

(A | Each) {Entity} {relationship} (one | many) {Entity} .

Example (1:M): A Student is assigned to many Courses.

Example (1:1): Each Student has a one Identification.
